                      Создание (старт проекта)

старт проекта с git flow, там же конфигурацию настраиваешь:
git flow init



                    Работа с фичами (FEATURE), заливаем изменения в ветку develop  (DEVELOP)
создать ветку с разработкой фичи:
git flow feature start editTxtFile

после того как фичу сделал закомить ей по дефолту, а после:
ты можешь запушить изменения в удаленный репо, чтобы твои коллеги могли посмотреть и поправить тебя:
git push origin feature/editTxtFile

после того, как коллеги одобрили можешь пушить изменения в ветку develop с помощью команды:
git flow feature finish editTxtFile

после этого изменения запушились в ветку develop, но они не отправились в ветку develop удаленного репо, для этого запушь изменения в develop:
git push origin develop

поздравляю! твои изменения успешно создал фичу и залил её в ветку develop удаленного репо, теперь все видят твои изменения.
После того, как все фичи будут сделаны и проект будет готов к релизу, можно его и сделать! 

                        (Release, Production)  Релиз, заливаем изменения в основную ветку в продакшн так сказать.
начинаем релиз (полагаю следует находиться в ветке develop) - git flow release start firstRelease
в этот момент откроется текстовый файл в котором можно(нужно!) указать теги для данного релиза, внедрили в проект слайдер? добавь тег slider, переделали авторизацию? тогда добавь тег auth и т.д.
после завершения просто сохрани ctrl+s и закрой текстовый файл, а после выполни команду для завершения релиза:
git flow release finish 'firstRelease'
